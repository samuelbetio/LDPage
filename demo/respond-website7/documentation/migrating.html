Migrating from Respond 5 to Respond 6

Respond is a complete overhaul of the system, but you can transition a site created in R5 to R6. In fact, we transitioned this site to use the new system.

1. Create a new site in R6 using the Blank template

Start by creating a new, blank site in R6. This will give you the structure you need to get started. We recommend using the theme that best matches your current theme or the basic Bootstrap theme to start. If you previously purchased a theme in R5, just send me an email at matt@matthewsmith.com, and I will send you the current version (if available).

2. Copy your R5 site to R6

Copy the contents of your R5 site to your R6 sites folder (e.g. sites/my-site). Do not copy the following folders: js, components, data, locales, templates, and themes.

Next give Respond permission to write to your site

// make the directories writeable
chown -R www-data app/public/sites
chown -R www-data app/resources/sites

3. Login and Run the migration developer tool

Login to your Respond site and run the migration development tool. Learn more Development tools. This does a lot of the heavy lifting for you.

4. Re-index your site

Re-index your site using the Re-index pages developer tool. Learn more Development tools.

5. Add references to the R6 plugins JS and CSS

Add references to the R6 plugins JS and CSS.

// head of document
<link type="text/css" href="css/plugins.css" rel="stylesheet">

// end page
<script src="js/plugins.js"></script>

6. Update/Add your menus, forms, galleries from the UI

Plugins will not copy over from R6. So, I recommend simply re-creating these using the R6 UI.
